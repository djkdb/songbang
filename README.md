# 🎤 셋리. — 노래방 랜덤 뽑기

> 노래방에서 "뭐 부르지?" 하며 인기차트를 한참 뒤지는 시간, 이제 끝!
> 내가 부를 수 있는 노래를 저장해두고, 버튼 한 번으로 랜덤으로 뽑아주는 노래방 도우미 앱입니다.

## ✨ 주요 기능

### 🎰 랜덤 뽑기
- **내 노래 / 인기차트 / 둘 다** 중에서 뽑기 소스 선택
- 장르 필터 (발라드, 댄스, 락/밴드, 힙합, R&B, 트로트, OST, 포크/인디, 팝)
- **오늘 이미 뽑은 곡 제외** 옵션 — 같은 곡이 또 나오는 것 방지
- 슬롯머신 애니메이션으로 두구두구 🥁 하는 재미
- 최근 뽑은 노래 기록 (최대 30곡)

### 🎤 내 노래 목록
- 제목 / 가수 / **TJ·금영 노래방 번호** / 태그 저장
- 검색, 수정, 삭제
- JSON 파일로 **내보내기·가져오기** (기기 옮길 때 유용)
- 모든 데이터는 브라우저 localStorage에 저장 — 서버 없이 동작

### 🔥 인기차트 100 (매월 자동 업데이트)
- **TJ미디어 노래방 TOP100(가요)** 을 GitHub Actions가 매월 긁어와 자동 갱신
- 제목·가수 검색, 장르 필터
- ♥ 버튼 한 번으로 내 노래에 담기 — 이때 **TJ 곡번호가 자동으로 채워져요**
- 차트에서 뽑힌 곡도 바로 내 노래에 추가 가능
- 네트워크가 없거나 첫 배포 전이면 앱 내장 큐레이션 차트로 자동 폴백

### 📱 PWA (앱처럼 설치)
- 홈 화면에 추가하면 앱처럼 실행
- 서비스 워커로 **오프라인에서도 동작** — 지하 노래방에서 신호가 안 터져도 OK

## 🚀 실행 방법

빌드 과정이 없는 순수 HTML/CSS/JS 앱이에요.

```bash
# 아무 정적 서버로 열면 됩니다
npx serve .
# 또는
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속. (`index.html`을 직접 열어도 동작하지만, PWA/서비스 워커는 http(s) 환경에서만 활성화됩니다.)

### GitHub Pages 배포
저장소 설정 → Pages → 브랜치 선택만 하면 바로 배포됩니다. 별도 빌드 설정이 필요 없어요.

## 📁 프로젝트 구조

```
songbang/
├── index.html                    # 앱 진입점 (뽑기/내 노래/인기차트 탭)
├── css/style.css                 # 뉴트로 노래책 테마 스타일
├── js/
│   ├── app.js                    # 앱 로직 (뽑기, 목록 관리, 저장, 라이브 차트 로드)
│   └── chart-data.js             # 내장 인기차트 100 (오프라인 폴백 & 장르/연도 룩업)
├── data/chart.json               # 자동 업데이트되는 인기차트 (Actions가 갱신)
├── scripts/update-chart.mjs      # TJ미디어 TOP100 스크래퍼 (의존성 없음, Node 20+)
├── .github/workflows/update-chart.yml  # 매월 자동 실행 워크플로우
├── icons/                        # 앱 아이콘 (SVG)
├── manifest.webmanifest          # PWA 매니페스트
└── sw.js                         # 오프라인 캐시 서비스 워커
```

## 🔄 인기차트 자동 업데이트 구조

노래방 차트의 **공식 오픈 API는 없어서**, TJ미디어 웹사이트(`/chart/top100`)가 내부적으로
사용하는 엔드포인트를 이용합니다.

```
매월 1일 ─▶ GitHub Actions ─▶ scripts/update-chart.mjs
                              (POST tjmedia.com/legacy/api/topAndHot100)
                                     │
                                     ▼
                              data/chart.json 커밋
                                     │
                        앱이 fetch 해서 인기차트 탭에 표시
                        (실패 시 js/chart-data.js 내장본으로 폴백)
```

- **수동 실행**: 저장소 → Actions → "인기차트 자동 업데이트" → *Run workflow*
- **주의**: GitHub의 스케줄 트리거는 **기본 브랜치에 워크플로우 파일이 있을 때만** 동작합니다.
  이 브랜치를 기본 브랜치로 병합해야 매월 자동 실행돼요.
- 스크래핑 대상 사이트의 이용약관을 준수하세요. 사이트 개편으로 응답 형식이 바뀌면
  `scripts/update-chart.mjs`의 `ENDPOINT`·필드 매핑(`indexTitle`/`indexSong`/`pro`)을 손보면 됩니다.
- 카테고리 변경: `TJ_STR_TYPE` 환경변수 (`1`=가요[기본], `2`=팝, `3`=J-POP)

## 📝 참고

- TJ 차트는 장르/연도를 제공하지 않아, 내장 차트(`js/chart-data.js`)와 제목·가수가 일치하는
  곡은 장르/연도를 자동 보강합니다. 신곡은 장르가 비어 있을 수 있어요(‘전체’에서는 항상 뽑혀요).
- TJ 차트에는 곡번호(`pro`)가 있어서, ♥로 담으면 **TJ 노래방 번호가 자동으로 채워집니다.** 금영 번호는 직접 입력해 주세요.
